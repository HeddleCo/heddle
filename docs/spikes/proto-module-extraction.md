# Spike: extract the gRPC contract into a dedicated API repository

Status: superseded by ADR 0048. This document is the pre-cutover extraction
spike and is retained as historical evidence. The accepted candidate direction
is a net-new `heddle.api.v1alpha1` contract in `HeddleCo/api`, not an unchanged
relocation of `heddle.v1`; the coordinated implementation remains blocked by
ADR 0048's checklist.

## Why now

The gRPC contract is a shared artifact consumed by the Heddle CLI and local
daemon, the Weft server and tools, and the Tapestry web product. Keeping its
source, Rust/TypeScript generation, and publication pipeline inside Heddle
couples all three products to Heddle's release cadence and crate graph.

The pressure is no longer hypothetical:

- `heddle-grpc` advanced from 0.13 to 0.22 between 2026-07-07 and 2026-07-12.
- Rust and TypeScript publication are split across Heddle's crate, stable-tag,
  and manually dispatched npm workflows.
- Wire compatibility is reviewed by hand; no Buf breaking-change gate exists.
- Before this preflight, the canonical entrypoint, Rust compiler inventory,
  idempotency audit, and TypeScript root exports could describe different
  subsets of the schema.

## Preflight baseline after `heddle-grpc` 0.22

- Canonical schema: `crates/grpc/proto/heddle/v1/` — 17 services and 176 RPCs
  under package `heddle.v1` after restoring every Weft-served method and
  removing only the unserved `TreeEditService` methods.
- Rust SDK: `heddle-grpc` 0.22, generated by `crates/grpc/build.rs` with
  tonic/prost, including `heddle_descriptor.bin` and explicit byte-backed
  fields.
- TypeScript SDK: `@heddleco/grpc` 0.22, generated with
  `@bufbuild/protoc-gen-es` under `clients/grpc`.
- In-tree Rust consumers: `crates/client`, `crates/cli`, and `crates/daemon`
  use path dependencies with a published-version requirement.
- Out-of-tree consumers: Weft uses the Rust packages; Tapestry uses the
  TypeScript package.

The schema contains both Heddle-local daemon contracts and Weft-hosted
contracts. A neutral API repository is therefore a better ownership seam than
either product repository.

The consumer evidence and pending producer cutovers are recorded in
`docs/spikes/proto-consumer-inventory.md`. ADR 0045 owns the durable repository,
contract-metadata, and cutover decisions.

## Locked decisions

### 1. Repository: `HeddleCo/api`

The repository owns the `.proto` source, compatibility policy, code generation,
descriptor artifact, and multi-SDK publication. It does not own Heddle or Weft
implementations.

### 2. Clean-first extraction

Heddle first makes `service.proto` a complete canonical entrypoint, derives
compiler and SDK inventories from the flat schema package, and derives
TypeScript root exports from the same inventory. Every method explicitly
declares its effect and deduplication contract. A descriptor audit verifies the
method options and the singular, typed request field marked as its retry key.
Extraction starts only after those checks are green.

This preflight removes only surfaces proven unimplemented before the first
API-repository release. Production handlers are a compatibility boundary; their
contracts remain until a coordinated consumer and producer cutover removes
them.

### 3. Tooling: adopt Buf without changing Rust generation during the move

The API repository adds `buf lint` and `buf breaking` against the last published
contract. The first release keeps the current tonic/prost build configuration
and TypeScript generator versions so repository relocation and generator
replacement cannot obscure each other's parity failures. A later change may
evaluate Buf-driven or checked-in Rust generation independently.

### 4. Publication: one tag drives both SDKs

CI builds and verifies both SDK artifacts before publishing either. A release
tag publishes `heddle-grpc` to crates.io and `@heddleco/grpc` to GitHub Packages.
Publication is automated and retryable; Heddle's manual npm dispatch is retired
after cutover.

### 5. Versioning: separate redesign from relocation

Do not reset to 0.1. Heddle publishes the coordinated identity and pruning
contract as 0.23. After every consumer uses it, `HeddleCo/api` imports that
frozen source unchanged and publishes 0.24. Registry availability is checked
immediately before each tag.

### 6. Consumers use published artifacts

Heddle, Weft, and Tapestry consume registry packages. Do not add a submodule or
vendored proto mirror. Committed consumer manifests point at released artifacts.

## Executable migration

### Phase -1 — clean the source contract in Heddle

- Make `service.proto` import every schema file.
- Enforce one flat `heddle.v1` inventory for Rust and TypeScript generation.
- Declare every RPC's effect and deduplication semantics with protobuf options.
- Audit those options and marked idempotency fields through compiled
  descriptors, including singular traversal through streaming envelopes.
- Generate and check TypeScript root exports from the same proto inventory.
- Refresh documentation and run Rust generation, contract audit, TypeScript
  generation/typecheck, and package smoke checks.
- Publish coordinated Rust and TypeScript SDK 0.23 from Heddle and cut Heddle,
  Weft, and Tapestry to it.

Exit gate: every contract consumer inside Heddle sees the same schema inventory,
adding a proto file without entrypoint/export coverage fails CI, and all three
products consume 0.23.

### Phase 0 — stand up `HeddleCo/api`

- Copy the frozen 0.23 proto tree and current Rust/TypeScript generator setup.
- Add Buf configuration, compatibility gates, protected release permissions,
  and package ownership.
- Record source-tree, descriptor, generated Rust, generated TypeScript, and
  package checksums.
- Build clean candidate packages without changing consumers.

Exit gate: candidate outputs match Heddle SDK 0.23, including
service paths, descriptor contents, byte mappings, request-signing bytes, and
public SDK exports.

### Phase 1 — publish API SDK 0.24

- Build, test, and pack both SDKs before publication.
- Publish Rust, verify a clean downstream build, then publish TypeScript and
  verify clean root/subpath imports.
- If either artifact is wrong, do not cut consumers over; roll forward with a
  new version rather than overwriting an immutable registry release.

### Phase 2 — cut consumers over

- Weft updates the direct `heddle-grpc` dependency and audits whether `weftctl`
  needs `heddle-client` or only the generated contract.
- Weft makes descriptor-to-registered-handler coverage fatal and resolves every
  uncertain producer entry in the consumer inventory.
- Heddle changes `client`, `cli`, and `daemon` from path dependencies to the
  published crate and publishes any new `heddle-client` version Weft requires.
- Tapestry updates `@heddleco/grpc` after the Weft producer is deployed.
- Run old-client/new-server and new-client/old-server tests for unary,
  server-streaming, and bidirectional sync paths.

Exit gate: all three repositories are green using published API artifacts and no
production consumer reads schema source from Heddle.

### Phase 3 — transfer ownership and delete Heddle copies

- Move schema/SDK compatibility checks to the API repository.
- Keep hosted handler/idempotency implementation checks in Weft and local-daemon
  behavior tests in Heddle; Heddle does not source-scan Weft.
- Delete `crates/grpc`, `clients/grpc`, Heddle's gRPC generation command, manual
  npm workflow, crate publication entry, and release-script assertions.
- Update Heddle documentation to link to the canonical API repository.

Exit gate: one small additive API release has flowed through the independent
publish-and-consume path, then the deletion PR passes all Heddle checks.

## Verification gates

- `buf lint` and `buf breaking` against the last published schema.
- Canonical entrypoint and full-tree inventory equality.
- Normalized descriptor equality; raw descriptor equality where deterministic.
- Generated Rust parity plus downstream client/server compile fixtures.
- Preservation of tonic byte-backed field configuration and
  `FILE_DESCRIPTOR_SET`.
- Request-signing and idempotency-hash known vectors across old/new SDKs.
- Generated TypeScript tree parity, complete root exports, typecheck, pack, and
  installed-package smoke imports.
- Heddle client/daemon/CLI tests, Weft server/tool tests, and Tapestry build.
- Trial additive change succeeds; trial breaking change is rejected.

## Deferred design work

Do not combine extraction with package `heddle.v1` restructuring, local/hosted
schema partitioning, SDK feature profiles, pre-generated Rust, or final Rust
operation interfaces. Those are follow-up deepening candidates after the
publication seam is proven.

Further package restructuring, local/hosted schema partitioning, and generator
replacement remain independent changes after the authority-transfer seam is
proven.
