# Sley Facade Roadmap

Status: inventory-only Wave 1 planning note. This document records Sley facade
gaps that Heddle callers currently work around. It does not change behavior,
does not describe shipped Sley APIs unless a current caller already uses them,
and must not be used as permission to add more Heddle-local Git plumbing.

Current dependency snapshot: `Cargo.toml` depends on `sley = "0.3.1"` with the
`mmap` and `fast-sha1` features. `Cargo.lock` pins the Sley crate family to
`0.3.1` from crates.io.

Consumption rule: prefer consuming facade work through a released Sley version
and then updating `Cargo.toml` / `Cargo.lock`. A path patch is acceptable only
for short-lived Heddle validation branches when it is explicitly called out
below; remove the patch before merging release-bound Heddle work unless the
maintainer asks otherwise.

Priority legend:

- `P0`: blocks correctness or risky mutation cleanup.
- `P1`: unlocks major cleanup, performance, or shared transfer planning.
- `P2`: ergonomic or future-feature unblocker; current thin adapter can remain.

Stop column:

- `Yes`: blocked Heddle work should stop at the facade gap and move to Sley
  instead of adding a local substitute.
- `Partial`: Heddle may keep or add a small temporary adapter for existing
  behavior, but larger cleanup should wait for Sley.
- `No`: current Heddle adapter is acceptable until normal Sley release uptake.

## Adoption Gates

When any row moves from workaround to Sley facade consumption, the Heddle change
should include the smallest relevant subset of these gates:

- Sley facade parity tests named in the row.
- Heddle integration tests for the listed callers.
- `cargo test --locked -p heddle-cli --test git_process_lint -- --nocapture`
  if any Git-shaped workaround is removed or reclassified.
- The Sley hardening tests listed in `docs/SLEY_INTEGRATION.md` when bridge,
  tags, remote config, or Git-overlay status behavior changes.

Do not add a Heddle-local Git plumbing substitute when a row marked `Yes` is the
actual blocker. Stop at the facade gap, file or implement the Sley-side API, and
then come back through the release/path-patch path above.
